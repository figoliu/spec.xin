## 1. 翻译检测系统详解

### 1.1 检测功能概述
翻译检测系统(`translation-detect`)能够智能识别项目中需要翻译的文件和翻译质量问题，为自动翻译提供精确指导。

### 1.2 核心检测能力

#### 环境验证和文件扫描
- 验证spec-kit目录存在且为最新版本
- 扫描所有需要翻译的目录: templates/, memory/, docs/, 根目录md文件
- 识别新增、修改或翻译缺失的文件
- 生成文件状态映射表

#### 翻译状态智能分析
- 完全未翻译(内容与原版相同)
- 需要更新(原版更新了内容, 没有同步翻译)
- 部分翻译(中英文混合)
- 翻译质量问题(不符合翻译标准)
- 格式问题(Markdown格式损坏、链接失效)

#### 优先级分析机制
- 核心模板 > 文档 > 辅助文件
- 基于文件重要性和修改频率的动态优先级

### 1.3 检测报告内容
- 翻译完成度统计
- 质量问题分类汇总
- 推荐的翻译策略
- 风险评估和注意事项

## 2. 完整翻译工作流程

### 2.1 核心命令体系
```
translation-auto      # 全自动翻译(一键式)
translation-sync      # 同步原版更新
translation-detect    # 检测翻译需求
translation-execute   # 执行翻译任务
translation-qa        # 质量保证检查
translation-fix       # 修复翻译问题
translation-review    # 人工审核
```

### 2.2 场景化工作流程

#### 首次设置翻译项目
```bash
/translation-detect    # 检测需要翻译的文件
/translation-auto      # 执行完整翻译流程
/translation-qa        # 质量检查
/translation-fix       # 修复发现的问题
```

#### 原版更新后的同步
```bash
/translation-sync      # 同步原版更新
/translation-qa        # 验证同步质量
/translation-fix       # 修复同步问题
```

#### 日常翻译维护
```bash
/translation-detect    # 检测新增翻译需求
/translation-execute   # 执行翻译
/translation-qa        # 质量检查
```

#### 发布前质量检查
```bash
/translation-qa        # 全面质量检查
/translation-fix       # 修复关键问题
/translation-review    # 最终人工审核
```

## 3. 翻译质量保障机制

### 3.1 翻译检测的质量评估标准
- 参考 @TRANSLATION_STANDARDS.md 和 @TERMINOLOGY.md 进行评估
- 识别机器翻译痕迹, 标记需要人工优化的内容
- 考虑文件间的依赖关系, 确保翻译一致性

### 3.2 多层质量控制策略
1. **自动化检测**: 使用智能对比算法, 不仅检查内容差异, 还要分析翻译质量
2. **重点内容审核**: 重点关注CLI相关文件的翻译准确性
3. **结构化输出**: 输出结构化的任务清单, 便于自动化处理

## 4. 高级工作流配置

### 4.1 定制化检测规则
- 可配置检测目录和文件模式
- 支持自定义优先级规则
- 提供翻译质量阈值设置

### 4.2 批量操作优化
- 并行处理多个翻译任务
- 智能缓存减少重复工作
- 增量更新策略节省资源

## 5. 团队协作最佳实践

### 5.1 基于检测结果的任务分配
- 利用translation-detect生成的优先级清单进行任务分配
- 针对不同复杂度的翻译任务分配适当的团队成员

### 5.2 协作流程标准化
1. **需求分析**: 使用 `translation-detect` 分析翻译需求
2. **任务分配**: 基于检测结果分配翻译任务
3. **并行执行**: 使用 `translation-execute` 并行处理
4. **质量检查**: 使用 `translation-qa` 统一质量标准
5. **问题修复**: 使用 `translation-fix` 处理质量问题
6. **最终审核**: 使用 `translation-review` 人工审核

## 6. 性能优化建议

### 6.1 检测效率提升
- 增量扫描减少处理时间
- 智能缓存提高重复检测速度
- 并行处理提升大规模项目检测效率

### 6.2 翻译执行优化
- 利用检测结果精确翻译, 避免全量处理
- 建立翻译记忆库, 提高一致性和速度
- 智能批量处理减少系统资源占用

## 7. 故障排除指南

### 7.1 常见检测问题及解决方法
- 文件路径错误: 检查目录结构和路径配置
- 翻译状态误判: 调整检测阈值或手动标记状态
- 性能问题: 减少扫描范围或调整并行度

### 7.2 工作流执行问题排查
- 命令顺序错误: 严格按照推荐工作流程执行
- 权限问题: 确保有足够权限访问和修改文件
- 依赖缺失: 检查所有必要工具是否正确安装

- 建立完善的翻译知识管理体系, 持续优化翻译标准
- 确保团队协作的高效性和一致性
- 定期回顾和改进翻译流程, 基于实际使用反馈调整策略
